  PHASE 6 — CENTRAL KITCHEN  [COMPLETE — 3 May 2026]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  One restaurant is designated a Central Kitchen (production hub). Other
  restaurants link to it and place distribution requests. Stock is deducted
  from the CK on dispatch and credited to the receiving restaurant on receive.

SCHEMA ADDITIONS  (migrations: 20260503000002_add_restaurant_type,

                                 | RECEIVED | CANCELLED

  [x] DistributionRequest model — fromRestaurantId, centralKitchenId, status,
        neededBy, notes, createdById
  [x] DistributionItem model — requestId, ckIngredientId, ingredientName, unit,
        requestedQty, approvedQty, dispatchedQty, receivedQty

WORKFLOW (DRAFT → SUBMITTED → APPROVED → DISPATCHED → RECEIVED)
  DRAFT       — created by linked restaurant, editable
  SUBMITTED   — locked, pending CK review
  APPROVED    — CK set approved qty per item
  DISPATCHED  — CK deducts stock (StockTransaction OUT) + records dispatchedQty
  RECEIVED    — receiving restaurant increments stock (auto-creates ingredient
                  if not found, StockTransaction IN) + records receivedQty

API ROUTES
  [x] GET  /api/central-kitchen/links     — context-aware: CK sees linked
        restaurants, restaurant sees its CKs + available CKs to link
  [x] POST /api/central-kitchen/links     — create link (restaurant side)
  [x] GET  /api/central-kitchen/requests  — CK: incoming requests;
        restaurant: outgoing requests (with items + statuses)
  [x] POST /api/central-kitchen/requests  — create distribution request with items
  [x] POST /api/central-kitchen/requests/[id]/approve   — set approvedQty per item
  [x] POST /api/central-kitchen/requests/[id]/dispatch  — deduct CK stock,
        record StockTransaction OUT, status → DISPATCHED
  [x] POST /api/central-kitchen/requests/[id]/receive   — increment restaurant
        stock (auto-create ingredient if missing), StockTransaction IN,
        status → RECEIVED
  [x] GET  /api/central-kitchen/demand    — CK only: aggregated demand from
        SUBMITTED+APPROVED requests grouped by ingredient, shows shortfall
  [x] GET  /api/central-kitchen/ingredients?ckId=X — list a CK's ingredients
        (accessible by CK members and their linked restaurants)
